IDE和目录结构

  • assets目录存放所有资源

    • res //导入资源
    • resources //代码导入资源
    • texture //纹理资源
    • script //js文件
    • scene //场景文件

  • .meta文件：类似管理文件，方便cocos的管理

    • 分配了一个uuid
    • 记录了一些参数信息

    大部分省略，详见官方文档

  • 将JS编辑器改为VS CODE

  • 预览运行：设计分辨率，适应屏幕宽度高度等等

COCOS 生命周期函数

  • onLoad：加载时
  • start：第一次Update之前，第一次渲染完成之后调用
  • update：游戏每次刷新调用
  • lateUpdate: 刷新后
  • onDestroy：销毁时调用
  • onEnable：启用时
  • onDisable：停用时

CC.NODE 属性

  • cc.node.name:

    • type：string
    • explain:节点的名称

  • cc.node.active:

    • type: boolean
    • explain: 是否被激活===是否显示

  • cc.node.position:

    • type: cc.v2(x: value,y: value)
    • explain: 相对父节点的坐标

  • cc.node.rotation:

    • type: number
    • explain: 旋转，顺时针为正，逆时针为负

  • cc.node.scale:

    • type: cc.v2(x:value, y:value)
    • explain: 缩放放大的数值

  • cc.node.anchor:

    • type:number
    • explain:锚点，坐标的中心, 上1，下0，左0，右1，左下角为(0,0)，右上(1,1).依次类推.锚点数值可以小于0，也可以大于1

  • cc.node.size:

    • type: cc.v2(x:value,y:value)
    • explain: 控制节点大小

  • cc.node.color:

    • type: color
    • explain: 环境光颜色，图片本身有颜色的话会覆盖一层红光在上面

  • cc.node.Opacity:

    • type:number(0-1)
    • explain: 透明度,0透明，1不透明

  • cc.node.skew:

    • type: cc.v2(x: value, y: value)
    • explain: 扭曲节点

  • cc.node.Group:

    • type: 组别

    • explain: 分组，设置节点所属组别以确定可否进行碰撞检测

      在项目设置分组管理中可以直接添加分组，添加分组，然后在每个节点的Group属性处都可以选择所属分组。

  • cc.node.parent: 父节点

  • cc.node.children: 孩子节点数组

  • cc.tag: 节点标签

cc.Component 介绍

  • 所有组件都扩展自cc.Component类

  • 包含node属性，方便索引组件所在的节点

    在cc.Class里面通过this.node即可访问得到组建的节点

  • 包含name属性，获得节点的名称

  • 包含enabled属性，用来控制组件的启动情况

  • 组件部分生命周期函数:

    • onLoad
    • start
    • update
    • lateUpdate
    • onEnabled
    • onDisabled

节点的动态操作

  • 添加: this.node.addChild(new cc.Node())

  • 添加节点到指定位置: this.node.insertChild()

  • 节点的删除: this.node.removeChild(node, cleanup)

  • 节点的清空:this.node.removeAllChildren(cleanup:boolean)

  • 子节点自己删除:node.removeFromParent(); 从父节点处删除调用子节点

  • 节点的搜索:

    • getParent
    • getChildByUuid
    • getChildByName
    • getComponent: 获取节点上指定类型的组件
      • getComponents
      • getComponentInChildren
      • getComponentsInChildren
    • find(Path):全场景查找，根据路径去查找

  • setLocalZOrder: 改变绘制的顺序,其实就是setZindex,通过代码动态更改zindex

    这个方法已经废弃，直接操作zIndex即可

关于脚本文件中编写cc.Class

  • 所有的脚本都是写一个cc.Class,在cc.Class中定义了一系列内容，这些内容在组件被绑定到节点上时被new了出来，我们实际填写的是构造函数。

AudioSource组件使用

  • 如何创建

    • 创建AudioSource节点
    • 在节点下创建AudioSource组件

  • 如何添加

    将音频文件拖到Clip处,支持wav,ogg,mp3文件格式

  • 参数调节

    • Clip: 音频剪辑
    • Volume: [0,1]的值，表示音量大小的百分比
    • Mute: 静音
    • Loop: 循环
    • Play on load: 加载时播放，勾选择音源在onEnable时播放
    • Preload：预加载

  • 代码操作AudioSource的属性

    onLoad阶段无法代码操作属性,因为onLoad阶段cocos会将编辑器页面的属性导入到AudioSource组件中，在onLoad阶段后，对AudioSource的属性操作就会生效

动画编辑器使用

Cocos creator自带的动画可以制作轻量的UI动画

  • Animation组件

    它是一个普通的组件，用来给节点添加动画效果

  • 添加动画效果？

    是通过creator编写Clip文件

  • 编辑动画的流程

    1. 创建节点
    2. 添加Animation组件
    3. 在动画编辑器编辑动画Clip
    4. 把动画设置到Animation组件

  • 属性动画制作

    1. 对节点添加属性动画
    2. 插入属性关键帧
    3. 在动画末尾插入关键帧
    4. 保存

  • 帧动画：同理，修改spirteFrame即可

  • 脚本控制动画

    • 关联动画组件

    • 操作动画

    • Animation其它属性

      • defaultClip: 默认动画
      • currentClip:当前动画

    • Animation播放事件

      • play
      • stop
      • pause
      • resume
      • lastframe：假如动画循环次数大于 1，当动画播放到最后一帧时
      • finished：动画播放完成时

      事件的监听需要由animation对象的on函数完成

  • 在动画中添加函数(事件帧)

    点击编辑器的“插入帧事件”即可添加事件帧。双击小白快即可进入编辑页面，填写方法名和参数。触发时Cocos会遍历节点的脚本组件去匹配这个方法。

cc.loader使用

Cocos有三种Pipeline

  • assetloader

  • downloader

  • loader: 这个loader用来加载游戏资源

loader的加载全部都是异步的

  • 加载本地资源

      1. loadRes只能加载resources文件夹下的文件

TIPS

  • assets目录下需创建res和resources两个目录，其中res是用来拖动添加的，而resources目录里的资源是代码添加的。如果不这么做，会导致项目打包之后setting.js文件异常的大。

    cc.loader.loadRes方法默认从resources目录下读取文件
